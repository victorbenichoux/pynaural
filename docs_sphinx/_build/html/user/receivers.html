Receivers¶

This documentation deals with the basic acoustical objects in the spatializer.

class spatializer.receivers.HeadReceiver(height, iad, orientation=Vector (0.0, 1.0, 0.0))[source]¶

Initialized with the height of the head center and interaural distance. Is a simple two-points receiver that represents an empty head.

** Initialization **

getSphericalBeam(n)[source]¶

Returns a spherical beam with 2*n rays, the first half coming from the left ear and the scond from the right ear.

class spatializer.receivers.OrientedReceiver(position, orientation)[source]¶

OrientedReceiver is just a regular receiver but oriented. This provides the reference for the azimuth and elevation in the HRTF computations

placeSource(distance, rotation)[source]¶

Yields a source at a certain distance, az and el relative to the oriented receiver object.

class spatializer.receivers.HRTFReceiver(position, orientation=Vector (0.0, 1.0, 0.0), is_distancedependent=False)[source]¶

A single-point receiver that uses HRTF to finally filter IRs. The position is the head center.

** Initialization **

position can be a Quantity or numeric object. If so the Receiver is positioned at position*UP. Alternatively it can be just a Vector or Point giving the real position.

`` orientation = FRONT `` Orientation of the Receiver object.

** Methods **

** Advanced **

An HRTFReceiver has to implement the ``get_hrir’’ method. ``get_hrir(az, el [,distance])’’ returns the hrir pair for the given direction.

Note that if the ``distance’’ argument is supported by ``get_hrir’’ then the ``is_distancedependent’’ property must be set to True.

class spatializer.receivers.IRCAMSubjectReceiver(height, orientation=Vector (0.0, 1.0, 0.0), subject=None)[source]¶

Is used to place a know subject of the IRCAM database in a scene. Initialized with the subject number, height, inter aural difference and orientation. You must instanciate the IRCAMpath preference variable for it to work.

get_normalization_factors()[source]¶

Returns the normalisation factors alphaleft alpharight so that all the filters on the left (resp. right) have a gain equals to 1.

The gain of a filter being defined as the sqrt of the sum of the square of the amplitudes of the tf.
